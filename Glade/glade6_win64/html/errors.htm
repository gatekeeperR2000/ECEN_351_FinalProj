Glade Reference

Error reporting

In the unfortunate event of an internal program error occurring, on all platforms except Solaris, Glade will trap the error and report diagnostics which can be mailed to the developers so the bug can be fixed. To get diagnostics reported, you must set the environment variable GLADE_NO_EXCEPTION_HANDLER to yes. Otherwise the exception may be handled by the default GUI exception handler.

The error report shows the type of error, CPU register contents and a stack trace with the most recent stack frame first. Clicking on the Mail bug report button will open up your system's default mailer (on Windows), and put a copy of the error report in the system clipboard. You can then paste the report into the mail. Clicking on the Exit button will exit the application. If there is unsaved data, you will be prompted to save the library.
